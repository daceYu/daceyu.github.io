制作Web站点时考虑的一些因素

May 26, 2016 | 前端开发移动端用户体验 | Hits

考虑 Web 应用的 UI

Web 应用的 UI 是用户使用应用的窗口，它的展现直接影响用户对于应用的感觉和认可度。为了不让用户失望而放弃使用，我们应该如何考虑 Web 应用的 UI 。这有点像产品经理应该考虑的事情，不过作为前端工程师也应该考虑这些事情。

1.界面美观大方，并且符合产品本身的风格
2.考虑大多用户的使用习惯，增强应用的易用性

  • 减少用户输入，而是采用提供选择的方式
  • 提供网站搜索功能
  • 给一些功能增加文字提示
  • 提供一些操作的快捷键
  • 增大功能热区区域，方便用户使用
  • 必要时，提供页面导航功能

3.增强页面的交互性

  • 比如鼠标移动到按钮上，按钮透明度或颜色发生变化
  • 搜索查询时，提供 loading 动画
  • 改进内容或状态展现的方式

4.错误提示对用户友好
5。保证应用的响应速度

考虑 Web 应用的安全性

Web 应用的 UI 是用户使用应用的窗口，它的展现直接影响用户对于应用的感觉和认可度。为了不让用户失望而放弃使用，我们应该如何考虑 Web 应用的 UI 。这有点像产品经理应该考虑的事情，不过作为前端工程师也应该考虑这些事情。

  1. 对输入进行有效验证。防止 XSS 攻击、 CSRF 攻击和 SQL 注入。
  2. 对一些交互操作进行权限验证。
  3. 异常错误时，不在页面上暴露服务器的信息。
  4. 浏览器控制台把调试的 console.log 语句删除。可以写一个方法来调 console.log 的方法，然后发到正式环境时返回空。
  5. 数据加密存储。

考虑 Web 应用的性能

(1)Content 方面

  • 合并文件，减少 HTTP 请求，压缩文件，减少资源大小。
  • 减少 DNS 查询次数，平衡 DNS 查询和并行下载的速度。
  • 非必须组件延迟加载。
  • 非首屏图片延迟加载。
  • 优化 DOM 结构，减少 DOM 元素数量。

(2)Server 方面

  • 使用 CDN 托管静态资源。
  • 使用缓存缓存资源。
    添加 Expires 或者 Cache-Control 响应头设置缓存。
    对资源配置 ETag。

(3)Cookie 方面

  • 减少 cookie 大小。
  • 引入资源的域名不要包含 cookie。

(4)CSS 方面

  • 将样式表放在 head 标签内。
  • 不使用 @import。
  • 图表使用 iconfont。
  • 可根据需要适当开启 transition3d 加速。
  • 压缩 CSS 文件。

(5)JavaScript 方面

  • 优化代码结构，提高代码解析速度。
  • 减少 DOM 访问。
  • 合理设置事件监听。
  • 删除没有作用的代码。
  • 压缩 JS 文件。

(6)图片方面

  • 优化图片，根据实际的颜色需要选择图片格式和压缩比率。
  • 不要拉伸图片。
  • 使用 favicon.ico 尽量小并且进行缓存。
  • 将比较小的图片转换成 base64 编码。

(7)移动端保证组件大小尽量小。

考虑 Web 应用的 SEO

1.合理的设置 title、description、keywords，搜索引擎对三者的权重依次减小。

  • title 值强调重点即可，不同页面 title 有所不同。
  • description 把页面内容高度概括，长度合适，不同页面 description 有所不同。
  • keywords 列出重要关键词即可。

2.语义化的 HTML 代码，符合 W3C 规范。语义化嗲吗让搜索引擎容易理解网页。
3.重要内容 HTML 代码放在最前。搜索引擎抓取 HTML 顺序从上到下，有的搜索引擎对抓取长度有限制，保证重要内容一定会被抓取。
4.重要内容不要使用 JS 输出。爬虫不会执行 JS 获取内容。
5.少用 iframe。搜索引擎不会抓取 iframe 中的内容。
6.非装饰性图片必须加 alt。
7.提供网站速度。网站速度是搜索引擎排序的一个重要指标。

考虑 Web 应用的可维护性

  • 考虑 Web 应用的可维护性，大体从三个方面来考虑，它们分别是开发、构建、部署。

开发

  • 有统一的编码规范，比如统一缩进格式、变量命名、书写规范。
  • 依赖管理，使用 bower、npm 工具来管理包依赖。
  • 项目组件化。整个 Web 应用有各个组件组成。
  • 模块化开发。JS 模块化、CSS 模块化、HTML 模块化，所有资源都当成模块。模块之间可以相互引用，达到重用模块的功能。
  • 根据需要文件实时监听，自动刷新浏览器。

构建

  • 搭建脚手架进行初始化构建。
  • 可以编译中间语言。比如 Less、Sass、TypeScrript。
  • 可以对文件进行压缩和混淆。
  • 有工具自动处理 CSS Sprite，方便开发和维护 CSS Sprite。
  • 生成文件时，加上 hash 值，方便精确控制缓存。

部署

  • 根据打包配置文件，打包不同环境的资源文件。
  • 打包工具可配置直接把静态文件上传至 CDN。

考虑 Web 应用的技术因素

  • 主要是根据项目的实际需要来选择合适的技术框架。
